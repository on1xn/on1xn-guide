# 🤝 Contribution Guidelines

To contribute to this project, you will need to have **Python**, **MkDocs** (To preview your changes in the browser), and **Git** installed on your system. 

If you prefer using a Python virtual environment, you can install all necessary dependencies using the `requirements.txt` file located in the project repository. (Alternatively, just installing Python and MkDocs manually works too!)

To get started, simply clone this repository, add your accessible mods or gameplay guides, and push your changes. Please note that all guides must be written as **Markdown (.md)** files.

> **Don't know how to use Python, MkDocs, Git, or GitHub?** 
> No worries! You can send your mods and guides directly to me through any of the platforms listed in the repository or on the guide pages. I'll format and convert them for the site.
> 
> Naturally, full credit will be given to all contributors on the website for their shared mods and guides!

## 📝 Markdown Style Guide

To maintain a consistent and highly readable layout across the entire project, please format your Markdown files according to these guidelines:

* **Heading Hierarchy:** All guides must start with a Heading 1 (`#`) for the guide's title. Use Heading 2 (`##`) through Heading 4 (`####`) for subsequent sub-sections.
* **Section Separators:** End each major section with a horizontal rule (`---`) to create a clean visual separation.
* **Bullet Points:** Use asterisks (`*`) to create bulleted lists. This applies to both mod lists and step-by-step instructions.
* **Mod Links:** Format all mod names and their corresponding download links as: `**[Mod Name](Download Link)**`.
* **Spacing in Lists:** For the accessible mods list, please insert two blank lines between each bullet point to keep the text spacious and easy to read.

* **Bold Emphasis:** While not strictly mandatory, feel free to use bold text (`**text**`) to highlight key terms or emphasize instructions (e.g., **Do this: X, Y, Z**). It helps keep our styling uniform, especially since AI tools love using this format, and we want to keep things consistent!
